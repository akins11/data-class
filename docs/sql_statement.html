3.0.1 SELECT

The SELECT statement in SQL is used to retrieve data from one or more tables in a database. The SELECT statement allows you to specify exactly which columns of data you want to retrieve, as well as to filter, sort, and group the results.

SQL
-- Basic Syntax
SELECT column_name1, column_name2, ...
FROM tableName;
1
The names of columns to retrieve (include in the result).
2
The name of the table to retrieve the records from.
Including the FROM clause

“The FROM clause is essential in SQL queries to explicitly specify the table you want to retrieve data from. It’s a fundamental component of the SELECT statement.

Multiple ways of selecting columns

You have the flexibility to select individual columns, retrieve all columns using the wildcard (*), or utilize expressions and functions to process the data before retrieving them.

-- All columns

FROM tableName;

-- Specific columns
SELECT column_name1, column_name2

-- Functions

FROM tableName;

-- Expression
SELECT 'Title: ' || column_name1 || ', Author: ' || column_name2
FROM tableName;

3.0.2 WHERE

The WHERE clause is used to filter records returned by a SELECT, UPDATE, DELETE, or other SQL statements. It specifies a condition that each row must meet in order to be included in the result set.

SQL
-- Basic Syntax
SELECT column_name1, column_name2, ...
FROM tableName
WHERE condition;
1
Include the WHERE clause after the FROM clause.

Key components of the WHERE statement
  • The condition is a logical expression that evaluates to TRUE, FALSE, or NULL.
  • Only rows for which the condition is TRUE are returned or affected by the query.

2. Logical operators:

You can also used to combine multiple conditions by using a logical operator.

Keyword Description
AND Both conditions must be true
OR At least one condition must be true
NOT Negates a condition
Logic behind the logical operators
condition A Logical Operator condition B Output
TRUE AND TRUE TRUE
TRUE AND FALSE FALSE
FALSE AND TRUE FALSE
FALSE AND FALSE FALSE
TRUE OR TRUE TRUE
TRUE OR FALSE TRUE
FALSE OR TRUE TRUE
FALSE OR FALSE FALSE

In SQL, the AND and OR operators have equal precedence. You can use parentheses () to control the order of evaluation if needed. See Tip 3.1

AND:

The AND operator is used to combine two or more conditions, and it returns true only if all the combined conditions are true. If any condition evaluates to false, the entire expression will return false.

SQL
SELECT title, genre, publication_year
FROM Books
WHERE genre = 'Fiction' AND publication_year = 1960;
1
Filter records of books that have a fiction genre and published in 1960.
title genre publication_year
To Kill a Mockingbird Fiction 1960

In this example, the query retrieves records from the Books table where both conditions are satisfied: the book must have a Fiction genre and the year in which the book was published must be 1960.

NOT:

The NOT operator negates a condition. It returns true if the specified condition is false and vice-versa. This operator can be used with other logical operators for more complex queries.

SQL

FROM Books
WHERE NOT genre = 'Fiction';

Tip 3.1: Order of evaluation

Logical Operators Precedence:

  • NOT Operator: The NOT operator has the highest precedence among logical operators. Any expression involving NOT will be evaluated first before considering other logical operations.

  • AND Operator: The AND operator has a higher precedence than the OR operator but lower than the NOT operator. When both AND and OR are present in a query without parentheses, AND conditions will be evaluated before OR conditions.

  • OR Operator: The OR operator has the lowest precedence among these three logical operators. When used alongside AND or NOT without parentheses, OR conditions will be evaluated after all AND conditions have been processed.

Combining logical operators with parentheses

To control the evaluation order explicitly and avoid ambiguity when combining these operators, you can use parentheses. Parentheses allow you to group conditions together so that they are evaluated first according to your requirements.

SQL
SELECT * 
FROM Books
WHERE (genre = 'Fiction' OR supplier_id = 4) AND publication_year > 2000;


3. BETWEEN and NOT BETWEEN:

These clause are specifically useful when trying to filter within a range.

BETWEEN:

The BETWEEN operator in SQL is used to filter the result set within a specified range. It allows you to select values that fall between two specified values, which can be numeric, text, or date types. The key characteristic of the BETWEEN operator is that it is inclusive; this means that both the lower and upper bounds are included in the results.

The general syntax for using the BETWEEN operator is as follows:

SQL
SELECT column_name(s)
FROM tableName
WHERE column_name BETWEEN value1 AND value2;
  • value1: The starting value of the range.
  • value2: The ending value of the range.

Example:

SQL
SELECT title, publication_year
FROM Books
WHERE publication_year BETWEEN 1997 AND 2003;
1
Filter records of books that were published between 1997 and 2003.
title publication_year
Harry Potter and the Philosopher''s Stone 1997
The Da Vinci Code 2003
 
SQL

FROM OrderStatus
WHERE order_id BETWEEN 4 AND 7;

Other data types
  • Text values: When used with text values, it sorts alphabetically. For example, selecting record names between ‘A’ and ‘M’:
SQL

  • Date values: You can also use it for date ranges. For instance, selecting records between January 1st and January 31st:

3.0.3 ORDER BY

The ORDER BY clause in SQL Server is used to sort the result set of a query by one or more columns. Sorting can be done in ascending (ASC) or descending (DESC) order, and it can be applied to any data type that can be ordered.

SQL
-- Basic Syntax
SELECT column_name1, column_name2, ...
FROM tableName
ORDER BY column_name1 [ASC|DESC], column_name2 [ASC|DESC], ...;
Key components of the ORDER BY clause

  • Using the ASC keyword, you can sort a column in ascending (small - big or first - last)

  • You can sort a column in descending order using the DESC keyword (big - small or last - first)

  • By default including the ORDER BY clause in your query will sort the values in ascending order.

  • You can sort by multiple columns. The result set is sorted by the first column, and within each value of that column, it is sorted by the next column, and so on.

Sorting multiple columns:

When you sort using the ORDER BY clause with multiple columns, SQL sorts the rows first by the first column listed, then by the second column, and so on.
The order of the columns in the ORDER BY clause is very important because each subsequent column is only used to sort rows that are tied .

How it works:

  • SQL sorts all rows based on the values in the first column you specify and this sets the primary order of the rows.

  • If there are rows with the same value in the first column, SQL then sorts those rows by the second column.

  • The process continues for all other additional columns.

SELECT unit_price, sales
FROM OrderStatus
ORDER BY unit_price, sales;

3.2 Conditional statement: (CASE WHEN)

The CASE statement in SQL Server is a conditional expression that allows you to perform if-then-else logic. It is used to return different values based on specific conditions, and it can be used in SELECT, UPDATE, INSERT, and ORDER BY clauses, among others.

Simple CASE expression:

SQL
CASE expression | column_name
    WHEN value1 THEN result1
    WHEN value2 THEN result2
    ...  
    ELSE default_result
END
1
The expression to evaluate or a column name.
2
Values1 that the expression is compared against.
3
The result returned if the corresponding WHEN clause is true.
4
(Optional). The value returned if none of the WHEN conditions are met. If ELSE is not provided and no WHEN condition is met, NULL is returned.
5
To indicate and end of the code block.
  • CASE: This keyword initiates the conditional statement.
  • WHEN: Each WHEN clause specifies a condition to evaluate.
  • THEN: The result that will be returned if the corresponding WHEN condition is true.
  • ELSE: This optional clause provides a default value if none of the WHEN conditions are met.
  • END: This keyword signifies the conclusion of the CASE statement.
How It works

When a query containing a CASE statement is executed, SQL evaluates each condition sequentially from top to bottom:

  • If a condition evaluates to true, SQL returns the associated result and stops evaluating further conditions.

  • If no conditions are true and an ELSE clause is present, SQL returns the value specified in ELSE.

  • If no conditions are true and there is no ELSE clause, SQL returns NULL.

Example:

SQL
SELECT 
    product_id,
    CASE product_id
        WHEN 1 THEN 'Product One'
        WHEN 2 THEN 'Product Two'
        WHEN 3 THEN 'Product Three'
        WHEN 4 THEN 'Product Four'
        ELSE 'Unknown Product'
    END AS product_name
FROM OrderStatus;

Terminating the CASE WHEN block

Make sure you include the END keyword at the end of the conditional block.
