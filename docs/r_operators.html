13  Operators

13.1 Comparison operators

Comparison operators are essential tools that allow you to compare values and make logical decisions based on the results. These operators return a logical value (TRUE or FALSE) depending on the outcome of the comparison.

13.1.1 Equality Operators

Equal to ( == )

This operator checks if two values are equal. If they are equal, it returns TRUE, otherwise, it returns FALSE.

Example

R
10 == 10
1
Check if 10 equal to 10. 
[1] TRUE
R
"Hello" == "Hello"
1
Check if both strings the same. 
[1] TRUE

Not equal to ( != )

This operator checks if two values are not equal. It returns TRUE if the values differ and FALSE if they are the same.

Example

R
10 != 10
1
Check if 10 not equal to 10. 
[1] FALSE
R
TRUE != FALSE
1
Check if TRUE not equal to FALSE 
[1] TRUE

13.1.2 Relational operators

Greater than ( > )

This operator checks if the left operand is greater than the right operand.

Example

R
100 > 50
1
Check if 100 is greater than 50 
[1] TRUE

Greater than or equal to ( >= )

This operator checks if the left operand is greater than or equal to the right operand.
Example

R
100 >= 100
1
Check if 100 is greater than or equal to 100 
[1] TRUE

Less than ( < )

This operator checks if the left operand is less than the right operand. It returns TRUE if this condition holds true.
Example

R
100 < 50
1
Check if 100 is less than 50. 
[1] FALSE

Less than or equal to ( <= )

This operator checks if the left operand is less than or equal to the right operand.
Example

R
50 <= 120
1
Check if 50 is less than or equal to 120. 
[1] TRUE

IN ( %in% )

The %in% operator in R is a built-in infix operator used to determine if elements of one vector are present in another vector. It returns a logical vector indicating whether each element of the first vector matches any element of the second vector

Example

R
5 %in% c(3, 5, 7, 9)
[1] TRUE

This operator is particularly useful for value matching and can be applied to both numeric and character vectors.




13.2 Logical operators

Logical can operators combine multiple conditions and evaluate them:

AND ( & )

The & operator is known as the element-wise AND operator. It takes two logical vectors and compares them element by element. The result is a new logical vector where each element is TRUE if both corresponding elements from the input vectors are TRUE; otherwise, it is FALSE.

Example:

R
(10 > 5) & (5 < 50)
1
Check whether both left and right hand side is TRUE 
[1] TRUE

OR ( | )

The | operator functions as the element-wise OR operator. It also compares two logical vectors element by element but returns TRUE if at least one of the corresponding elements is TRUE. If both are FALSE, then it returns FALSE.

Example:

R
(10 > 100) | (20 <= 50)
1
Check whether any of the left and right hand side is TRUE 
[1] TRUE

Negation operator ( Not ! )

The ! operator represents logical negation NOT. It takes a single logical value and returns its opposite: if the input value is TRUE, it returns FALSE; if it’s FALSE, it returns TRUE.

Example:

R
!(10 > 5)
1
Negate the output of the value 10 > 5 
[1] FALSE

Special comparison operators

identical()

This checks whether two objects are exactly identical in terms of attributes and types.

obj_1 <- c(1, 2, 3)
obj_2 <- c(1, 2, 3)

identical(obj_1, obj_2)
[1] TRUE

13.3 Control Flow

13.3.1 IF ELSE

This is used to execute code conditionally based on whether one or more conditions are true. This structure allows you to make decisions within your code and perform different actions depending on the outcome of those conditions.

Basic Structure and Syntax

The structure allows for multiple conditions to be checked in sequence. The first true condition will have its corresponding block of code executed, and the rest will be skipped.

R
if (condition1) {
  # Code to execute if condition1 is TRUE
} else if (condition2) {
  # Code to execute if condition2 is TRUE
} else {
  # Code to execute if none of the above conditions are TRUE
}
  • if Statement: Checks the first condition. If it’s true, the code block inside the if statement is executed.
  • else if Statement: Checks the next condition if the previous if or else if was false. You can have multiple else if statements.
  • else Statement: Executes a block of code if none of the preceding if or else if conditions are true. The else statement is optional.
How it works

  • The if condition is evaluated first.

  • If the if condition is true, its block of code runs, and the rest (else if and else) are skipped.

  • If the if condition is false, the next else if condition is evaluated.

  • This process continues through all else if conditions.

  • If no if or else if conditions are true, the else block is executed.

Examples Basic Example

R
x <- 200

if (x < 10) {
  print("x is less than 10")
} else if (x > 200) {
  print("x is greater then 200")
} else {
  print("x is equal 200")
}
[1] "x is equal 200"

Using Multiple else if Statements

R
score <- 105

if (score >= 90) {
  grade <- "A"
} else if (score >= 80) {
  grade <- "B"
} else if (score >= 70) {
  grade <- "C"
} else if (score >= 60) {
  grade <- "D"
} else {
  grade <- "F"
}

print(grade)
[1] "A"

Nested if…else statements

if...else statements can be nested within each other, allowing for more complex decision trees.

Example:

R
x <- 5
y <- 10

if (x > 0) {
  
  if (y > 5) {
    
    print("x is positive and y is greater than 5")
    
  } else {
    
    print("x is positive and y is 5 or less")
    
  }
  
} else {
  
  print("x is zero or negative")
  
}
[1] "x is positive and y is greater than 5"

ifelse() for vectorized operations

For vectorized operations, ifelse() function is preferred over if...else because it works element-wise on vectors, matrices, or arrays.

Basic Syntax:

R
ifelse(test_expression_or_condition, value_if_true, value_if_false)

Example

R
ifelse(10 > 5, "Condition is true", "Condition is false")
1
Use ifelse() function to return a value based on the condition. 
[1] "Condition is true"

Nested option

R
x <- c(5, 15, 25)

ifelse(x < 10, "Low", ifelse(x < 20, "Medium", "High"))
1
Use ifelse() function to classify values. 
[1] "Low"    "Medium" "High"

Handling complex conditions

Conditions can use logical operators (&&, ||, &, |, !) to combine multiple tests within a single if statement.

R
age <- 30
income <- 60000

if (age > 25 && income > 50000) {
  
  print("Eligible for the loan")
  
} else if (age > 25 || income > 50000) {
  
  print("Partially eligible for the loan")
  
} else {
  
  print("Not eligible for the loan")
  
}
1
Check whether age is greater than 25 and income is greater than 50000 
[1] "Eligible for the loan"
Key points to remember

  • Condition evaluation: Conditions must return TRUE or FALSE. Non-logical values are coerced to logical (e.g., 0 is FALSE and non-zero numbers are TRUE).

  • Short-circuiting in && and ||: Use && and || for scalar comparisons and avoid evaluating unnecessary conditions for performance efficiency.

  • Element-wise operations: For element-wise operations on vectors, always use & and | instead of && and ||.



13.3.2 For loop

The for loop is used to repeat a block of code for a specified number of iterations, typically by iterating over elements in a sequence, vector, list, or any iterable object. It is a control structure that helps automate repetitive tasks, making your code more efficient and readable.

Basic Structure and Syntax

R
for (variable in sequence) {
  # Code to execute on each iteration
}

  • variable: This is the loop variable that takes on each value in the sequence one at a time.

  • sequence: This is the iterable object, such as a vector, list, or sequence, over which the loop iterates.

  • Loop Body: The block of code inside the braces { } that is executed on each iteration.

How the for loop works

  • The loop starts by initializing the loop variable to the first value in the sequence.

  • The code inside the loop body is executed.

  • The loop variable is then updated to the next value in the sequence.

  • Steps 2-3 are repeated until all values in the sequence have been processed.

Examples: #### Basic for loop iterating over a vector {.unnumbered}

R
for (i in 1:5) {
  print(i)
}

[1] 5

Iterating over a character vector

R
student_names <- c("Alice", "Bob", "Charlie")

for (name in student_names) {
  print(name)
}

2
Print the current student name for that particular sequence.

nested for loops

You can nest for loops within each other to perform more complex iterations, such as iterating over rows and columns of a matrix.

13.3.4 While loop

The while loop is used to repeatedly execute a block of code as long as a specified condition remains true. Unlike the for loop, which iterates over a sequence of values, the while loop relies on a condition and continues to loop until that condition is no longer met. It’s useful when the number of iterations is not known beforehand.

Basic structure and syntax

R
while (condition) {
  # Code to execute as long as the condition is TRUE
}

  • condition: A logical expression that is evaluated before each iteration of the loop. If it evaluates to TRUE, the loop body is executed. If it evaluates to FALSE, the loop terminates.

  • Loop Body: The block of code inside the braces { } that runs repeatedly as long as the condition is true.

How it works

  • Before each iteration, the condition is checked.

  • If the condition is true, the code inside the loop body is executed.

  • After executing the loop body, the condition is checked again.

  • The loop continues until the condition becomes false.

Examples

R
counter <- 1

while (counter <= 5) {
  print(counter)
  counter <- counter + 1
}

13.4 Functions

Functions are essential for organizing and reusing code, making your programming more efficient and modular. They allow you to encapsulate blocks of code that perform specific tasks, which can then be executed with different inputs as needed.

Basic Structure of Functions

Syntax for Creating a Function:

R
function_name <- function(arg1, arg2, ...) {
  # Code to execute
  # Return a value (optional)
}

# OR using the lambda function Syntax

function_name <- \(arg1, arg2, ...) {
  # Code to execute
}
  • function_name: The name of the function that you’ll use to call it later.
  • Arguments (arg1, arg2, …): Inputs to the function. These can have default values.
  • Function Body: The code inside the braces { } that defines what the function does.
  • Return Value: The result of the function, which can be explicitly returned using the return() function or implicitly as the last evaluated expression.

Example

R
# Function to add two numbers
add_numbers <- function(a, b) {
  summed_values <- a + b
  return(summed_values)
}

# Call the function
add_numbers(5, 3)
1
Add the values assigned to the arguments.
2
Return the summed value. 
[1] 8

13.4.1 Function arguments

  • Required arguments: Arguments that must be provided when calling the function.

  • Default arguments: Arguments that have default values, allowing the function to be called without specifying those arguments explicitly.

  • Named arguments: Arguments can be specified by name when calling the function, which is useful when you want to provide values out of order.

Example with default arguments:

R
multiply <- function(a, b = 2) {
  return(a * b)
}
1
Function with a default value of for the b argument
2
Return the multiplied value. 
R
# Calls with and without the second argument
multiply(5)
1
Uses default b = 2 to multiple the value 5. 
[1] 10
R
# Overrides default
multiply(5, 3)  
[1] 15

Named arguments

R
add_numbers(b = 3, a = 5)
[1] 8
