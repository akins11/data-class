9  Subquery & CTE

9.1 Subquery

A subquery, also known as a nested query or inner query, is a query within another SQL query. It allows you to construct complex queries by embedding one query inside another. Subqueries can be used in various parts of SQL statements, including SELECT, INSERT, UPDATE, and DELETE statements.

Key characteristics of subqueries

  • They are enclosed in parentheses.

  • They can be used in the SELECT, FROM, WHERE, HAVING, and JOIN clauses of the main query.

  • They can return a single value, a single row, a single column, or a table of results depending on how theyâ€™re constructed.

9.1.1 Scalar subquery

A scalar subquery returns a single value and can be used wherever a single value expression is allowed.

9.1.3 Correlated subquery

A correlated subquery is a subquery that depends on the outer query for its values and is executed once for each row in the outer query.

We can retrieve products with a price higher than the average price in their category: Subquery: To do this letâ€™s get the average price per unit for each category.

SQL
SELECT category, AVG(unit_price) AS average_unit_price
FROM Products
GROUP BY category;
1
Get the average unit price and give it the name average_unit_price.
2
Group the records by product category.

Full query: Now we can retrieve products with a unit price higher than the average for their category.

SQL
SELECT 
    p1.product_id, 
    p1.product_name, 
    p1.unit_price, 
    p1.category
FROM Products p1
WHERE p1.unit_price > (
    SELECT AVG(p2.unit_price)
    FROM Products p2
    WHERE p2.category = p1.category
);

The process

  1. It looks at each product (p1) in the outer query.
  2. For each product, it calculates the average price of products in the same category (p2).
  3. Then it compares the productâ€™s price to this average.
  4. If the productâ€™s price is higher than the average for its category, itâ€™s included in the results.

9.2 Common table expressions (CTEs)

CTEs are a feature in SQL that provide a way to create temporary result sets that can be referenced within a SELECT, INSERT, UPDATE, or DELETE statement. Unlike subqueries, CTEs make queries easier to understand and can be reused within the main query.

SQL
-- Basic syntax of a CTE
WITH CTE_name (column_name1, column_name2, ...) AS (
    -- CTE Query
    SELECT column_list
    FROM table_name
    WHERE condition
)
-- Main query using the CTE
SELECT column_name1, column_name2
FROM CTE_name
WHERE condition;
About the CTE query

  • WITH: This keyword is used to define a CTE. It tells SQL Server that a CTE is about to be declared.

  • CTE_name: This is the name of the CTE, it is is a temporary, virtual table name that can be used within the scope of the statement.

  • (column_name1, column_name2, ...): These are optional column aliases that specify the names of the columns in the CTE. If not specified, the columns from the inner SELECT query will be used by default.

  • AS (): This keyword specifies the beginning of the CTE definition.

9.2.2 Recursive CTE

Used for hierarchical data, like organizational structures or tree data.

SQL
WITH RECURSIVE RecursiveOrderDetails AS (
    -- Anchor member: Start with the first order detail
    SELECT 
        order_detail_id, 
        product_id, 
        quantity, 
        quantity AS cumulative_quantity
    FROM OrderDetails
    WHERE order_detail_id = 1  -- Start from the first OrderDetail (or any base case)

    UNION ALL

    -- Recursive member: Incrementally add quantities of the next order detail
    SELECT 
        od.order_detail_id, 
        od.product_id, 
        od.quantity, 
        rod.cumulative_quantity + od.quantity AS cumulative_quantity
    FROM OrderDetails od
    INNER JOIN RecursiveOrderDetails rod 
        ON od.order_detail_id = rod.order_detail_id + 1  -- Go to the next order detail
)
-- Main query using the CTE
SELECT 
    order_detail_id, 
    product_id, 
    quantity, 
    cumulative_quantity
FROM RecursiveOrderDetails
ORDER BY order_detail_id;

9.2.3 Multiple CTEs

Combining multiple CTEs in a single query.

Identify the highest unit price for each product category, and then list products with a unit price exceeding 80% of their respective categoryâ€™s highest unit price.

  1. Letâ€™s determine the highest unit price for each product category.
SQL
SELECT category, MAX(unit_price) AS max_unit_price_per_cat
FROM Products
GROUP BY category;
1

  1. Retrieve products with a unit price exceeding 80% of the maximum unit price within their respective product categories.
SQL
WITH cat_max_unit_price_cte AS (
    SELECT category, MAX(unit_price) AS max_unit_price_per_cat
    FROM Products
    GROUP BY category
)
SELECT 
    p.product_id, 
    p.product_name, 
    p.unit_price,
    c.category, 
    c.max_unit_price_per_cat,
    0.8 * c.max_unit_price_per_cat AS unit_price_80_percent
FROM Products AS p
INNER JOIN cat_max_unit_price_cte AS c
    ON p.category = c.category
WHERE p.unit_price > 0.8 * max_unit_price_per_cat;

  1. Select the product, category and unit price greater than 80% of the unit price in their category. NOTE that we can add more complex query from the resulting records of the high_products CTE
SQL
WITH cat_max_unit_price_cte AS (
    SELECT category, MAX(unit_price) AS max_unit_price_per_cat
    FROM Products
    GROUP BY category
),
high_products AS (
    SELECT 
        p.product_id, 
        p.product_name, 
        p.unit_price,
        c.category, 
        c.max_unit_price_per_cat
    FROM Products AS p
    INNER JOIN cat_max_unit_price_cte AS c
        ON p.category = c.category
    WHERE p.unit_price > 0.8 * max_unit_price_per_cat
)
SELECT 
    product_name, 
    category, 
    unit_price
FROM high_products;
