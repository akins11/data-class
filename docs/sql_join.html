8  Joins

JOIN statements in SQL are used to combine rows from two or more tables based on a related column between them. They are essential for retrieving data from multiple tables in a relational database.


Example Tables

  • Customer Table:
customer_id customer_name
1 Paul
2 Kelvin
3 John
4 David
  • Orders Table:
order_id customer_id order_date
101 1 2024-08-01
102 2 2024-08-02
103 1 2024-08-03
104 3 2024-08-04


8.1 INNER JOIN

INNER JOIN returns records that have matching values in both tables. This means that only the rows that meet the specified condition are included in the result set. If there are no matches, those rows are excluded.

SQL
-- Syntax
SELECT table1.id, table1.X_col, table2.Y_col
FROM table1
INNER JOIN table2
ON table1.id  = table2.id;
1
This specifies the primary table from which you want to retrieve data.
2
This indicates that you are performing an INNER JOIN operation with table2
3
This line specifies the condition for the join. The ON keyword is used to define how the two tables are related.

table1
id X_col
1 X1
2 X2
3 X3

table2
id Y_col
1 Y1
2 Y2
4 Y3

table1 INNER JOIN table2
id X_col Y_col
1 X1 Y1
2 X2 Y2

Example

Show customers who have orders.

SQL
SELECT 
    customers.customer_id, 
    customers.customer_name, 
    orders.order_id
FROM customers
INNER JOIN orders
    ON customers.customer_id = orders.customer_id;
customer_id customer_name order_id
1 Paul 101
1 Paul 103
2 Kelvin 102
3 John 104
Using Alias AS

The above join query can easily be written as follows:

SQL
SELECT c.customer_name, o.order_id
FROM customers AS c
INNER JOIN orders AS o
    ON c.customer_id = o.customer_id;




8.2 LEFT JOIN (LEFT OUTER JOIN)

This returns all records from the left table and the matched records from the right table. If there are no matches in the right table, the result set will still include all rows from the left table, but the columns from the right table will contain NULL values.

SQL
-- Syntax
SELECT table1.id, table1.X_col, table2.Y_col
FROM table1
LEFT JOIN table2
ON table1.id  = table2.id ;
1
This indicates the primary table from which the query will start retrieving data.
2
This specifies that you are performing a LEFT JOIN with table2. The LEFT JOIN keyword indicates that all records from the left table table1 will be included in the result, along with matching records from the right table table2.
3
This specifies the condition for the join operation. The id is a column that exists in both table1 and table2.

table1
id X_col
1 X1
2 X2
3 X3

table2
id Y_col
1 Y1
2 Y2
4 Y3

table1 LEFT JOIN table2
id X_col Y_col
1 X1 Y1
2 X2 Y2
3 X3 NULL

Example

Shows all customers, including those without orders (with NULLs in order_id).

SQL
SELECT 
    customers.customer_id, 
    customers.customer_name, 
    orders.order_id
FROM customers
LEFT JOIN orders
    ON customers.customer_id = orders.customer_id;
customer_id customer_name order_id

1 Paul 103

8.3 RIGHT JOIN (RIGHT OUTER JOIN)

This type of join is the opposite of the LEFT JOIN. It returns all records from the right table and the matched records from the left table. If there are no matches in the left table, the result will still include all rows from the right table, with NULLs in the columns from the left table.

SQL
-- Syntax
SELECT table2.id, table1.X_col, table2.Y_col
FROM table1
RIGHT JOIN table2
ON table1.id  = table2.id;
1
This indicates the primary table from which the query will retrieve the data.
2
This specifies that you are performing a RIGHT JOIN with table2. The RIGHT JOIN keyword indicates that all records from the right table table2 will be included in the result, along with matching records from the left table table1.
3
This specifies the condition for the join operation. The id is a column that exists in both table1 and table2.

id X_col
1 X1
2 X2
3 X3

id Y_col
1 Y1
2 Y2
4 Y3

table1 RIGHT JOIN table2
id X_col Y_col
1 X1 Y1
2 X2 Y2
4 NULL Y3

Example

Show all orders, including those without customers (with potential NULLs in customer_name).

SQL
SELECT 
    customers.customer_id, 
    customers.customer_name, 
    orders.order_id
FROM customers
RIGHT JOIN orders
    ON Customers.customer_id = orders.customer_id;

8.4 FULL JOIN (FULL OUTER JOIN)

The FULL JOIN combines the results of both LEFT and RIGHT joins. It returns all records from both tables, with matched records where available. If there is no match, the result will include NULLs for the columns of the table that does not have a corresponding entry.

SQL
-- Syntax
SELECT table1.id, table1.X_col, table2.Y_col,
FROM table1
FULL JOIN table2
ON table1.id  = table2.id ;
1
This indicates the primary table from which the query will retrieve the data.
2
This indicates that a FULL JOIN being performed with table2. The FULL JOIN keyword tells SQL to return all records from both tables involved in the join, regardless of whether there are matching records in the other table.
3
This specifies the condition for the join operation. The id is a column that exists in both table1 and table2.

table1
id X_col
1 X1
2 X2
3 X3

table2
id Y_col
1 Y1
2 Y2
4 Y3

table1 FULL JOIN table2
id X_col Y_col
1 X1 Y1
2 X2 Y2
3 X3 NULL
4 NULL Y3

Example

Show all customers and orders, including customers without orders and orders without customers.

SQL
SELECT 
    customers.customer_id, 
    customers.customer_name, 
    orders.order_id
FROM customers
FULL OUTER JOIN orders
    ON customers.customer_id = orders.customer_id;
customer_id customer_name order_id

8.5 CROSS JOIN

The CROSS JOIN produces a Cartesian product of the two tables, this means that it returns every possible combination of rows from both tables. This type of join does not require a condition to join the tables.

Cartesian product

Because CROSS JOIN does not require a condition to join the tables, this can lead to a large number of results, especially if both tables contain many rows.

SQL
-- Syntax
SELECT table1.id, table1.X_col, table2.id, table2.Y_col
FROM table1
CROSS JOIN table2

This indicates the primary table from which the query will retrieve the data.
2

8.6 SELF JOIN

This is a regular join but the table is joined with itself. Often used to compare rows within the same table.

SQL
-- Syntax
SELECT x.column_name, y.column_name
FROM table1 AS x, table1 AS y
WHERE x.id <> y.id;
1
Select columns from the two table1 using the separate table to distinguish between records.
2
This indicates the data is being selected from table1 twice only and an alias is created to distinguish between the two records.
3
This specifies the condition for the join operation. The id is a column that exists in both table1 and table2.

8.7 UNION

The UNION operator combines the results of two or more SELECT statements into one. It automatically removes duplicate rows from the final result set, ensuring that only unique records are displayed.

SQL
SELECT id, col
FROM table1
UNION
SELECT id, col
FROM table2;
Key important points about the UNION operator

  • The number and order of columns in all SELECT statements must match.

  • The data types of corresponding columns must be compatible.

  • UNION removes duplicate rows.

table1
id col
1 X1
2 X2
3 X3

table2
id col
1 X1
2 X2
4 X4

table1 UNION table2
id col
1 X1
2 X2
3 X3
4 X4

Example

SQL
SELECT product_id, product_name
FROM Products_A
UNION
SELECT product_id, product_name
FROM Products_B;
product_id product_name
1 Laptop
2 Smartphone
3 Tablet



8.8 UNION ALL

UNION ALL is similar to UNION but does not remove duplicate rows. This means it combines all rows from the queries, including duplicates.

SQL
SELECT id, col
FROM table1
UNION ALL
SELECT id, col
FROM table2;
