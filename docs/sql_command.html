2.2 Modifying a Table

There are several modifications that can be performed using SQL to change the structure of an existing table in a database, such as adding, modifying, or deleting columns, and setting constraints.

  1. Add a New Column:
SQL
-- Add a new column to an existing table.
ALTER TABLE tableName
  ADD column_name data_type;
  1. Drop a Column:
SQL
-- Delete an existing column from a table.
ALTER TABLE tableName
  DROP COLUMN column_name;
  1. Modify an Existing Column:
SQL
-- Change the data type of an existing column.
ALTER TABLE tableName
  ALTER COLUMN column_name new_data_type;
  
-- Example:
ALTER TABLE Orders
  ALTER COLUMN Discount DECIMAL(6, 2);
  1. Add Constraints:
SQL
-- Add a constraint e.g. (PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK)
ALTER TABLE tableName
  ADD CONSTRAINT constraint_name constraint_type (column_name);

-- Example:
ALTER TABLE Orders
  ADD CONSTRAINT PK_OrderID PRIMARY KEY (Order_ID);

2.3 Deleting a Table:

This is used to permanently delete a table from the database. When you execute the DROP TABLE command, the specified table and all of its data, structure, indexes, constraints, and permissions are completely removed from the database.

SQL
DROP TABLE tableName;
Warning

If the table has any constraints like foreign keys, you may need to drop those constraints first or use cascading options to automatically drop dependent objects.

SQL
-- Drop the foreign keys Constraint
ALTER TABLE MainTableName
  DROP CONSTRAINT FK_column_name;
  

-- The drop the table
DROP TABLE TableToBeDropped;

2.4 Querying Data

2.4.1 Selecting Data

The SELECT statement in SQL is used to retrieve data from one or more tables in a database. The SELECT statement allows you to specify exactly which columns of data you want to retrieve, as well as to filter, sort, and group the results.

SQL
SELECT column_name1, column_name2, ...
FROM tableName
Multiple ways of columns

You can select specific columns, use * to select all columns, or use expressions and functions.

SQL
-- All columns
SELECT *
FROM tableName

-- Specific columns
SELECT column_name1, column_name2
FROM tableName

-- Expression and functions
SELECT 
    COUNT(*),
    SUM(column_name)
FROM tableName

SELECT * FROM LibraryDB;

2.6 Inserting, Updating and Deleting Data:

2.6.1 Inserting Data

This is used to add new rows of data into a table.

Warning

If you want to insert values into all columns of a table, you can do so by specifying the values in the correct order. Or better still Insert data into specific columns

SQL
-- Basic Syntax
INSERT INTO tableName (column_name1, column_name2, ...)
  VALUES (value1, value2, ...);

-- Example
-- Insert sample data into Books table
INSERT INTO Books (title, author, genre, publication_year, isbn)
VALUES 
('To Kill a Mockingbird', 'Harper Lee', 'Fiction', 1960, '9780446310789');

You can also insert multiple rows separated by comma

SQL
INSERT INTO Books (title, author, genre, publication_year, isbn)
VALUES 
('1984', 'George Orwell', 'Science Fiction', 1949, '9780451524935'),
('Pride and Prejudice', 'Jane Austen', 'Romance', 1813, '9780141439518');

Lastly you can insert values from other similar table (also known as using a Subquery) with the same data type.

SQL
INSERT INTO tableName (column_name1, column_name2, ...)
SELECT column_name1, column_name2, ...
FROM OtherTableName;

2.6.2 Updating Data

This is used to change existing data in one or more rows of a table.

SQL
-- Basic Syntax
UPDATE tableName
  SET column_name1 = new_value1, column_name2 = new_value2, ...
  WHERE condition;


-- Example:
UPDATE Employees
  SET Salary = 65000
  WHERE EmployeeID = 1;
1
The condition can be based on one or more columns.
Important

The WHERE clause is optional, but if you omit it, the UPDATE statement will modify all rows in the table, which is often not the desired outcome.

SQL
UPDATE Employees
  SET Salary = Salary * 1.05;

2.6.3 Deleting Data

This is used to remove rows from a table. essentially it allows you to delete one or more rows based on a specified condition.

SQL
-- Basic Syntax:
DELETE FROM tableName
  WHERE condition;

-- Example:
DELETE FROM Employees
  WHERE EmployeeID = 1;
1
Specifies which rows should be deleted based on a condition relating to one or more columns.
Removing all Records

Always use a WHERE clause unless you intend to delete all rows in the table. Omitting the WHERE clause will result in the deletion of every row in the table, which may be irreversible without a backup.

SQL
-- Deleting All Rows:
DELETE FROM tableName;
